# 今天吃什么？- 美食转盘游戏 设计文档

**日期：** 2026-04-12
**版本：** v1.0

---

## 1. 项目概述

**项目名称：** 今天吃什么？（Eat If）
**项目类型：** Android 休闲游戏应用
**核心功能：** 通过各种小游戏随机决定用户下一顿饭吃什么
**目标用户：** 日常生活中纠结吃什么的人群

---

## 2. 功能架构

### 2.1 核心流程

```
启动APP → 主界面 → 选择模式（单人/双人）→ 选择游戏类型 → 游戏中 → 结果（美食决定）
```

### 2.2 游戏模式

| 模式 | 说明 | 决策方式 |
|------|------|---------|
| 单人模式 | 用户独自游玩 | 游戏结果停在哪，该选项即为吃饭选择 |
| 双人竞技 | 双人同屏对战 | 赢家的选项即为吃饭选择 |

## 3. 数据层设计

### 3.1 美食数据来源

**组合方案：美团API + 用户自建库**

| 来源 | 说明 | 优先级 |
|------|------|-------|
| 美团API | 实时获取当地餐厅/菜品数据 | 优先 |
| 用户自建库 | 用户手动维护的个人美食库 | 备用/补充 |

### 3.2 美团API集成

- 通过美团开放平台获取餐饮数据
- 支持按类别筛选（中餐/西餐/日料等）
- 数据缓存本地，减少API调用

### 3.3 用户美食库

**设置页面入口：** 我的 → 美食库管理

**支持操作：**
- 添加美食（名称、分类、图片）
- 编辑已有美食
- 删除美食
- 批量导入/导出
- 设置每道菜的权重（出现概率）

## 7. 游戏结果决定机制

### 7.1 单人模式

1. 用户选择游戏并开始
2. 游戏进行中，美食选项列表实时显示
3. 游戏结束时，根据游戏类型产生一个**停止位置/结果值**
4. 该位置对应的美食即为本次选择的"今天吃什么"

### 7.2 双人竞技模式

1. 两位玩家选择同一游戏
2. 游戏内设计PVP对战机制
3. 赢家从美食列表中**选择或随机决定**吃什么
4. 可选：输家有权否决一次（重新选择）

## 8. 非功能性需求

- **性能：** 游戏帧率 ≥ 30fps
- **兼容性：** Android 7.0 (API 24) 及以上
- **包体积：** 目标 ≤ 50MB（不包含游戏资源）
- **权限：** 网络权限（美团API）、存储权限（用户图片）
